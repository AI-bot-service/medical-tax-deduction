# Web UI — Современный фронтенд-интерфейс

Создай современный, оптимизированный, лёгкий веб-интерфейс промышленного уровня по запросу: $ARGUMENTS

---

## Принципы создания

### 1. Дизайн-система HEITKAMP (обязательно для этого проекта)

Все страницы и компоненты используют **HEITKAMP Design System**:

- Шрифт: **Urbanist** (Google Fonts, weights 300–800)
- Акцентный цвет: `#7B6FD4` (лавандово-фиолетовый)
- Фон страницы: `#F2F2F7`
- Фон карточек: `#FFFFFF`
- Тёмный sidebar: `#1A1A2E`
- CSS-переменные: `--accent`, `--bg`, `--surface`, `--sidebar-bg`
- Готовые классы: `.card`, `.btn-primary`, `.badge-*`, `.kpi-card`, `.sidebar`, `.nav-item`, `.topbar`

Перед созданием **прочитай файлы дизайн-системы**:
- `proekt/design/heitkamp/heitkamp-ds.css`
- `proekt/design/heitkamp/design-tokens-heitkamp.json`

Если файлы дизайн-системы существуют — используй их классы и переменные. Не переопределяй токены HEITKAMP своими цветами.

### 2. Технический стек проекта

Фронтенд проекта: **Next.js 15 App Router, TypeScript, Tailwind CSS, ShadCN/ui**.

- Для компонентов проекта — TypeScript + Tailwind
- Для изолированных артефактов (лендинги, постеры, демо) — допустимы чистый HTML/CSS/JS
- State: TanStack React Query + Zustand

### 3. Качество дизайна — требования

**Обязательно:**
- Уникальный визуальный язык — не "стандартный Bootstrap/MUI вид"
- Продуманная иерархия визуальных элементов (размер, вес, цвет, пространство)
- Micro-interactions: hover-состояния, transitions (150–300ms ease), subtle shadows
- Responsive: mobile-first, breakpoints sm/md/lg/xl
- Pixel-perfect отступы — система 4px/8px grid
- Тёмный режим: если уместно, поддержать через CSS custom properties

**Избегать:**
- Шаблонной "AI-эстетики" — стандартных градиентов hero-секций, банальных иконок
- Избыточных анимаций, отвлекающих от контента
- Захламлённых макетов — white space это дизайн
- `!important` без необходимости
- Инлайн-стилей вместо классов

### 4. Оптимизация и производительность

- Только используемые CSS-классы (не импортировать целые библиотеки для 1 компонента)
- SVG-иконки инлайн или через спрайт — не icon-fonts
- Изображения: `next/image` с `width`/`height`, `loading="lazy"` для off-screen
- Критический CSS — инлайн в `<head>` для above-the-fold контента (для HTML-артефактов)
- Минимум JS для статических страниц

### 5. Доступность (a11y)

- Семантический HTML: `<nav>`, `<main>`, `<section>`, `<article>`, `<button>` вместо `<div>`
- ARIA-атрибуты где нужны: `aria-label`, `aria-expanded`, `role`
- Контрастность текста: минимум WCAG AA (4.5:1 для нормального текста)
- Focus-visible стили для keyboard navigation
- Alt-тексты для всех значимых изображений

## Процесс создания

1. **Проанализируй запрос** — определи тип: компонент / страница / лендинг / дашборд / постер
2. **Проверь дизайн-систему** — прочитай `heitkamp-ds.css` и `design-tokens-heitkamp.json` если существуют
3. **Спланируй структуру** — layout, секции, компоненты
4. **Напиши код** — чистый, семантичный, с комментариями только там, где логика неочевидна
5. **Проверь**: responsive? accessible? performance? design quality?

---

## Форматы вывода

- **Next.js компонент**: `.tsx` файл в `frontend/src/components/` или `frontend/src/app/`
- **Изолированный HTML-артефакт**: один самодостаточный `.html` файл со встроенным CSS
- **Tailwind компонент**: JSX с Tailwind-классами, совместимыми с проектным `tailwind.config`

Создавай промышленный уровень — код, который не стыдно показать в портфолио.
