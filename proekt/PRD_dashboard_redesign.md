# PRD — Редизайн дашборда МедВычет

**Версия:** 1.1  
**Дата:** апрель 2026  
**Статус:** Draft  
**Область:** Frontend (дашборд) + Backend (новый API эндпоинт лимитов)

---

## 1. Контекст и цель

Текущий дашборд перегружен разрозненными панелями и не даёт пользователю понятной картины о его вычетах. Цель редизайна — сделать дашборд информативным, функциональным и визуально понятным. В перспективе дашборд должен поддерживать несколько типов вычетов (лечение, обучение, фитнес), а пока полностью сфокусироваться на «Социальном вычете».

**Принцип UI**: использовать функциональные, красочные и по ситуации анимированные иллюстрации/иконки вместо стандартных эмодзи.

## 2. Описание изменений по панелям

### Панель 1 — Левая верхняя (навигация / тип вычета)

**Текущее состояние:** надпись «МедВычет» с логотипом.  
**Требуемое изменение:**

- Убрать надпись «МедВычет», заменить на **«Социальный вычет»** с соответствующей иконкой.
- Добавить иконку-кнопку «i» (информация) с tooltip/popup следующего содержания:

  > «Если вы работаете официально и платите НДФЛ (13%), государство позволяет вернуть часть этого налога за расходы на лечение и обучение. Это и есть социальный вычет.»

- Под описанием — ссылка **«Подробнее →»**, при нажатии переводит на страницу `/info/social-vychet` с полным текстом из файла `Социального налогового вычета России.txt`.
- Навигация должна поддерживать **несколько типов вычетов** в будущем (список: Социальный вычет, Имущественный и т.д.). При выборе конкретного вычета — правая часть дашборда перерисовывается под него.

**Acceptance criteria:**
- Текст «МедВычет» заменён на «Социальный вычет».
- Иконка «i» показывает краткое описание при клике/hover.
- Ссылка «Подробнее» ведёт на страницу с полным описанием.
- Компонент рассчитан на добавление новых пунктов вычетов без переписывания.

### Панель 2 — Правая верхняя (фильтр по году)

**Текущее состояние:** отображается только текущий год (2026), без возможности переключения.  
**Требуемое изменение:**

- Отобразить **4 года**: текущий год и 3 предыдущих (например: 2023, 2024, 2025, 2026).
- Активный год выделен визуально (цвет/подчёркивание).
- При клике на год — **все панели дашборда перерисовываются** с данными за выбранный год (пересчёт сумм, лимитов, статистики).
- Список лет формируется динамически от текущего года.

**Acceptance criteria:**
- Отображаются 4 кнопки-года.
- Клик по году фильтрует данные всего дашборда.
- Активный год визуально отличается.
- При смене года данные в панелях 3 и 4 обновляются без перезагрузки страницы.

### Панель 3 — Лимиты и типы расходов по году

**Текущее состояние:** отсутствует как отдельная панель.  
**Требуемое изменение:**

Отобразить лимиты **в графическом виде** — блок из стаканов (вертикальные индикаторы наполнения). Таблица не используется.

**Визуальная концепция:**

Блок состоит из двух уровней:

**Уровень 1 — Большой общий стакан** (слева или по центру, шире и выше остальных):
Показывает совокупный лимит 150 000 ₽, в который входят «Лечение (обычное)» и «Обучение своё». Заливка внутри разделена на **две цветные зоны** — каждая зона соответствует своей категории. Цвета зон совпадают с цветами соответствующих малых стаканов.

**Уровень 2 — Малые стаканы** (справа от большого, меньшего размера):
Четыре стакана по категориям: «Лечение», «Обучение своё», «Обучение ребёнка», «Дорогостоящее лечение». Стаканы «Лечение» и «Обучение своё» окрашены в те же цвета, что и их зоны в большом стакане — визуальная связь между ними очевидна.

```
  ┌───────────────┐     ┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐
  │               │     │       │  │       │  │       │  │       │
  │               │     │       │  │       │  │       │  │       │
  │  ░░░░░░░░░░░  │     │░░░░░░░│  │       │  │       │  │░░░░░░░│
  │  ░░░░░░░░░░░  │     │░░░░░░░│  │░░░░░░░│  │       │  │░░░░░░░│
  │  [синий зона] │     │[синий]│  │[зелён]│  │░░░░░░░│  │  ∞    │
  │  [зелён зона] │     │       │  │       │  │[фиол] │  │       │
  └───────────────┘     └───────┘  └───────┘  └───────┘  └───────┘
   Общий лимит           Лечение   Обучение   Обучение   Дорогост.
   150 000 ₽            (обычное)   своё       ребёнка    лечение
   [лечение + обучение]  синий      зелёный    фиолетов.   ∞ / синий
```

**Цветовая схема категорий:**
- Лечение (обычное) → **синий** (`--blue` / `--tint-blue`)
- Обучение своё → **зелёный** (`--green` / `--tint-green`)
- Обучение ребёнка → **фиолетовый** (`--purple` / `--tint-purple`)
- Дорогостоящее лечение → нейтральный синий, символ ∞ без заливки

**Детали большого стакана (общий лимит):**
- Высота = (сумма расходов по лечению + обучению своё) / 150 000 ₽ × 100%.
- Внутри — две цветные зоны, пропорциональные каждой категории.
- Над стаканом — подпись «Общий лимит», общая сумма потрачено / 150 000 ₽, процент заполнения.
- Hover-tooltip: разбивка по зонам — сколько занято лечением, сколько обучением.

**Детали малых стаканов:**
- Высота заливки = расходы по категории / лимит категории × 100%. Если данных нет — стакан пустой.
- Цвет заливки — согласно цветовой схеме выше.
- Над стаканом — процент заполнения (например, «47%»), для безлимитных — прочерк.
- Под стаканом — название категории, лимит и максимальный возврат.
- Hover-tooltip: потрачено / лимит / возможный возврат.
- Для «Дорогостоящего лечения» (`is_uncapped = true`) — заливки нет, внутри анимированный символ ∞.

**Источник данных:**
- Лимиты берутся из нового API эндпоинта `GET /api/v1/tax-limits?year={year}`, который читает из `backend/app/services/deduction/limits.py` — новая БД-таблица не нужна.
- Расходы пользователя по каждой категории берутся из существующего `/api/v1/summary?year={year}`.
- Данные динамически обновляются при смене года в панели 2.

**Acceptance criteria:**
- Большой общий стакан визуально выделен (шире и выше малых), содержит две цветные зоны.
- Цвета зон большого стакана совпадают с цветами малых стаканов соответствующих категорий.
- Малые стаканы отображают заполненность согласно расходам пользователя.
- Для безлимитной категории — символ ∞ вместо заливки.
- Hover-tooltip на каждом стакане показывает точные цифры.
- При смене года все стаканы обновляются.

### Панель 4 — Прогресс сбора документов

**Текущее состояние:** панели 4, 5, 6 — статусы обработки, «Последние чеки» и «Быстрые действия».  
**Требуемое изменение:**

Удалить панели 4 (статусы), 5 (последние чеки) и 6 (быстрые действия). Создать **единую панель «Мои документы»** — обзор прогресса сбора всего пакета документов для налогового вычета.

Drag-and-drop не нужен (загрузка документов — отдельная страница). OCR-ошибок нет — каждый документ проходит ручную проверку пользователем.

**Группы документов и их содержимое:**

Панель отображает 5 групп документов, соответствующих категориям вычета (из PRD_SocialVychet_Lechenie_v1.md раздел 4):

| # | Группа | Что входит | Нужна при |
|---|--------|-----------|-----------|
| 1 | **Справки из клиник** | Справка КНД 1151156 (код 01 — обычное лечение, код 02 — дорогостоящее) | Любом лечении |
| 2 | **Рецепты** | Рецепт по форме № 107-1/у, 2-й экземпляр со штампом «Для налоговых органов РФ» | Вычете на лекарства |
| 3 | **Чеки из аптек** | Кассовые чеки при покупке лекарств | Вычете на лекарства |
| 4 | **Документы ДМС** | Справка КНД 1151159, договор ДМС, лицензия страховой | Вычете на ДМС |
| 5 | **Справки 2-НДФЛ** | Справка от работодателя за каждый год подачи | Классическом порядке (до 2024 г.) |

**Визуальное представление каждой группы:**

Каждая группа — горизонтальная карточка с:
- Иконкой типа документа слева
- Названием группы и кратким пояснением
- Двумя счётчиками справа: **«Загружено: N»** и **«На проверке: M»** (M — документы в статусе PENDING/REVIEW, ещё не подтверждённые пользователем)
- Статусным индикатором: ✅ если все нужные документы загружены и подтверждены, ⚠️ если чего-то не хватает или есть ожидающие, ○ если группа не используется (пользователь не добавил расходы этой категории)

**Пример отображения:**

```
┌─────────────────────────────────────────────────────────────┐
│  Мои документы                          Загружено / Ожидают │
├─────────────────────────────────────────────────────────────┤
│  🏥  Справки из клиник (КНД 1151156)        [3] / [1]   ⚠️  │
│      Справки с кодом лечения из медорганизаций              │
├─────────────────────────────────────────────────────────────┤
│  💊  Рецепты (форма № 107-1/у)              [5] /  0    ✅  │
│      Со штампом «Для налоговых органов РФ»                  │
├─────────────────────────────────────────────────────────────┤
│  🧾  Чеки из аптек                         [12] / [2]   ⚠️  │
│      Кассовые чеки при покупке лекарств                     │
├─────────────────────────────────────────────────────────────┤
│  🛡  Документы ДМС                           0  /  0    ○   │
│      КНД 1151159, договор, лицензия страховой               │
├─────────────────────────────────────────────────────────────┤
│  📄  Справки 2-НДФЛ                         [1] /  0    ✅  │
│      От работодателя за каждый год                          │
└─────────────────────────────────────────────────────────────┘
```
`[N]` — кликабельная ссылка. `0` без скобок — не кликабельна.

**Логика переходов по кликам на счётчики:**
- **Число «Загружено» > 0** → клик ведёт на `/receipts?type={group}&status=confirmed&year={year}` — все подтверждённые документы этой группы.
- **Число «Ожидают» > 0** → клик ведёт на `/receipts?type={group}&status=pending&year={year}` — документы, ожидающие проверки пользователем.
- **Число = 0** → не кликабельно, отображается как обычный текст без стиля ссылки.
- Кнопки «Перейти к документам» нет — переход только через цифры.

**Acceptance criteria:**
- Отображаются 5 групп документов.
- Счётчики «Загружено» и «На проверке» соответствуют реальным данным из базы за выбранный год.
- Статусный индикатор (✅ / ⚠️ / ○) корректно отражает состояние группы.
- Клик по группе ведёт на страницу документов с нужным фильтром.
- При смене года данные обновляются.

## 3. Новый API эндпоинт лимитов

Существующий `backend/app/services/deduction/limits.py` уже содержит все данные по лимитам для всех лет. Новая таблица в БД не нужна — добавляем только эндпоинт, который сериализует эти данные для фронтенда.

### Эндпоинт

```
GET /api/v1/tax-limits?year=2024
```

### Логика на бэкенде (`backend/app/routers/tax_limits.py`)

Для заданного года функция читает из `limits.py`:
- `get_aggregate_limit(year)` → совокупный лимит (например, 150 000 ₽)
- `get_child_education_limit(year)` → лимит на ребёнка (например, 110 000 ₽)
- `UNCAPPED_CATEGORIES` → категории без лимита
- `AGGREGATE_CATEGORIES` → категории в совокупном лимите
- НДФЛ-ставка → из `ndfl.py` `get_marginal_rate(year, income=0)` (базовая ставка 13%)

### Формат ответа

```json
{
  "year": 2024,
  "combined_limit": 150000,
  "limits": [
    {
      "type_key": "treatment_regular",
      "type_name": "Лечение (обычное)",
      "limit_amount": 150000,
      "refund_amount": 19500,
      "is_uncapped": false,
      "is_separate": false
    },
    {
      "type_key": "education_self",
      "type_name": "Обучение своё",
      "limit_amount": 150000,
      "refund_amount": 19500,
      "is_uncapped": false,
      "is_separate": false
    },
    {
      "type_key": "education_child",
      "type_name": "Обучение ребёнка",
      "limit_amount": 110000,
      "refund_amount": 14300,
      "is_uncapped": false,
      "is_separate": true
    },
    {
      "type_key": "treatment_expensive",
      "type_name": "Дорогостоящее лечение",
      "limit_amount": null,
      "refund_percent": 13,
      "is_uncapped": true,
      "is_separate": true
    }
  ]
}
```

Логика отображения на фронтенде: если `is_uncapped = true` → показывать **∞** и `refund_percent`%, иначе — `limit_amount` ₽ и `refund_amount` ₽.

## 5. Нефункциональные требования

- Переключение года — без перезагрузки страницы (React Query invalidation).
- UI компоненты используют существующий стек: Next.js 15, Tailwind CSS, ShadCN/ui.
- Иконки и иллюстрации — из Lucide Icons или кастомные SVG, не эмодзи.
- Эндпоинт `/api/v1/tax-limits` — публичный (без авторизации), аналогично `/calculator/simple`.

## 6. Вне scope (не делаем сейчас)

- Поддержка других типов вычетов кроме «Социального» (только заготовка в навигации).
- Мобильная адаптация дашборда (отдельная задача).
- Редактирование лимитов через UI — при необходимости обновлять `limits.py` и деплоить.
